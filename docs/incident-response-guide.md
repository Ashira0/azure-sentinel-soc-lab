# Incident Response Guide

This document outlines the investigation workflow for alerts generated by the Microsoft Sentinel detection rules implemented in this lab.

---

# Detection 1: Brute Force Login Attempts (Event ID 4625)

## Description
This alert detects multiple failed authentication attempts against a single account from the same IP address within a short time window.

## Investigation Steps
1. Identify the affected account from the alert entity.
2. Review the source IP address generating the failed logins.
3. Check the number of failed attempts and time interval.
4. Determine whether the IP address belongs to a legitimate user.

## Validation Queries

Example investigation query:
Event
| where EventID == 4625
| summarize FailedAttempts=count() by AccountName, IpAddress
| order by FailedAttempts desc


## Containment Actions

If malicious activity is confirmed:

- Temporarily disable the affected account
- Block the attacking IP address
- Reset the user password

# Detection 2: Failed Logins Followed by Successful Login

## Description
This detection identifies a sequence of multiple failed login attempts followed by a successful authentication for the same account.

This pattern may indicate a brute force attack that eventually succeeded.

## Investigation Steps

1. Identify the affected account.
2. Review the timeline of failed login attempts.
3. Confirm whether a successful login occurred shortly after the failures.
4. Verify whether the login source IP is expected.

## Validation Queries
Event
| where EventID in (4624,4625)
| summarize count() by EventID, Computer


## Containment Actions

- Reset the compromised account password
- Revoke active sessions
- Review additional activity performed by the account

---

# Detection 3: Account Lockout (Event ID 4740)

## Description

This alert triggers when an account becomes locked due to repeated failed authentication attempts.

Account lockouts may indicate password spraying or brute force activity.

## Investigation Steps

1. Identify the locked account.
2. Determine the originating computer or IP.
3. Review authentication attempts leading up to the lockout.
4. Confirm whether the activity was legitimate.

## Validation Queries
Event
| where EventID == 4740
| project TimeGenerated, Computer


## Containment Actions

- Reset the locked account password
- Investigate the source of failed attempts
- Review security logs for additional suspicious activity
